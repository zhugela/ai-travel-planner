## 一、概念题（背诵版）

### 1. 大模型 4 种分类方式，各举一例


| 分类维度     | 含义           | 举例                                               |
| -------- | ------------ | ------------------------------------------------ |
| **按模态**  | 看输入/输出是什么类型  | 文本：**Qwen-Plus**；多模态：**Qwen-VL**（图文）             |
| **按开源性** | 权重是否公开、能否自部署 | 闭源 API：**GPT-4**；开源：**Qwen2.5**                  |
| **按规模**  | 参数量大小        | 小模型：**Gemma3:1b**（本地）；大模型：**Qwen-Max**           |
| **按用途**  | 面向什么任务设计     | 对话：**Qwen-Plus**；Embedding：**text-embedding-v3** |


**一句话记忆**：模态看输入，开源看能不能自己部署，规模看参数量，用途看干什么活。

---

### 2. 「云服务 vs 自部署」的区别


| 对比项       | 云服务                                | 自部署                 |
| --------- | ---------------------------------- | ------------------- |
| **怎么用**   | 调厂商 API（如百炼 DashScope）             | 本机/服务器跑 Ollama、vLLM |
| **模型从哪来** | 厂商托管，始终较新                          | 自己下载开源权重            |
| **费用**    | 按 token 按量付费                       | 硬件 + 电费，无 API 费     |
| **数据**    | 请求上云                               | 可完全留在内网             |
| **能力**    | 强（Qwen-Max 等）                      | 受本机 GPU/内存限制        |
| **本项目**   | `SdkAiInvoke` / `SpringAiAiInvoke` | `OllamaAiInvoke`    |


---

### 3. 「平台 vs 程序接入」的区别

这里对比的是接入大模型的 **三条路径** 里和开发相关的两条：


| 对比项     | 平台接入（AI 应用平台） | 程序接入                   |
| ------- | ------------- | ---------------------- |
| **在哪做** | 百炼网页控制台       | IDEA 写 Java 代码         |
| **谁来做** | 产品/运营也能配      | 开发者                    |
| **形式**  | 拖拽智能体、工作流、群组  | SDK / HTTP / Spring AI |
| **优点**  | 快、可视化、免写代码    | 可定制、可集成业务、可上线          |
| **缺点**  | 难做复杂业务逻辑      | 要编码、要运维                |
| **关系**  | 先平台验证 Prompt  | 再程序产品化                 |


**补充**：还有「客户端接入」（ChatGPT App、通义 App），那是终端用户直接用，不是开发集成。
