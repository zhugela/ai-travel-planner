# 一、什么是智能体?

## 1.1 定义

**智能体(Agent) = 以大语言模型为核心**,融合记忆 / 知识库 / 工具调用能力的 AI 系统。

**vs 普通 LLM 的 6 大区别**:

| # | 能力 | 普通 LLM | 智能体 |
|---|---|---|---|
| 1 | 感知环境 | ❌ | ✅ |
| 2 | 自主任务拆解 | ❌ | ✅ |
| 3 | 主动调用工具 | ❌ | ✅ |
| 4 | 多步链式推理 | ❌(单轮) | ✅(CoT) |
| 5 | 长期记忆 | ❌(单次) | ✅ |
| 6 | 反馈动态调优 | ❌ | ✅ |

**日常生活例子**:**DeepSeek「深度思考」** = 轻量化智能体能力 — 先分步推理,再出答案,不是直接给。

## 1.2 三层分类(自主性递增)

# 二、智能体实现关键技术

4 大底层支撑技术,是所有自主智能体的通用基础。

## 2.1 CoT 思维链(Chain of Thought)

**作用**:让 AI 模仿人类**分步思考**,处理复杂推理问题时,先拆解推导,再输出答案。

**2 种落地方式**:
1. **基础 Prompt 引导**:`请一步一步思考问题`
2. **Few-shot 少样本**:给模型看带完整思维过程的示例

**OpenManus 例子**:
```python
Thinking: 详细拆解推理过程
Answer: 最终精简答案
```

## 2.2 Agent Loop 执行循环(智能体最核心运行机制)

**定义**:**脱离用户实时输入**,智能体自主循环执行「推理→工具→结果」,直到任务完成或达最大步数。

**vs 传统 LLM**:
- 传统 LLM:一问一答,单次交互
- Agent Loop:单次输入后,AI 自主多轮工具调用,**无需用户再干预**

**Java 伪代码**:
```java
while(currentStep < MAX_STEPS && !isFinished()){
    currentStep++;
    String stepResult = executeStep();
}
```

## 2.3 ReAct 模式(Reasoning+Acting)— **工业界主流**

4 步核心循环:
1. **Reason 推理**:拆解任务,确定下一步动作
2. **Act 行动**:调用外部工具(浏览器 / 代码 / 搜索 / DB)
3. **Observe 观察**:捕获工具返回结果
4. **循环迭代**:回到 Step 1,直到任务完成

**Java 演示代码**:
```java
void executeReAct(String task) {
    String state = "开始";
    while (!state.equals("完成")) {
        String thought = "思考下一步行动";
        String action = "执行具体操作";
        String observation = "观察执行结果";
        state = "完成";
    }
}
```

## 2.4 4 大支撑系统

| 系统 | 作用 |
|---|---|
| **AI 大模型** | 智能体大脑 |
| **记忆系统** | 存对话历史 + 中间状态 |
| **知识库(RAG)** | 弥补静态知识局限 |
| **工具调用系统** | 跳出自有模型,访问外部能力 |

**特殊工具**:**Compute Use** = AI 直接操控本机(代码 / 文件 / 系统),⚠️ **必须在沙箱/虚拟机跑**,防止误删文件。

# 四、四层 Agent 核心源码逐段解析

## 4.1 BaseAgent 基础层(模板方法设计模式)

```python
# 主循环 while
while current_step < max_steps and state != FINISHED:
    current_step += 1
    step_result = await self.step()  # 子类实现
    if self.is_stuck():
        self.handle_stuck_state()  # 防死循环
```

**职责**:
- 管理 IDLE / RUNNING / FINISHED
- 读写记忆
- 限最大步数
- 抽象 `step()`,子类实现「思考+行动」

## 4.2 ReActAgent 第二层(落地 ReAct 思想)

```python
async def think(self) -> bool: ...   # 推理:要不要行动
async def act(self) -> str: ...      # 行动:调工具/输出
async def step(self):
    should_act = await self.think()
    if not should_act: return "思考完毕,无需操作"
    return await self.act()
```

**单步固定流程**:先推理,再行动。**不**实现工具调用,仅流程骨架。

## 4.3 ToolCallAgent 第三层(框架核心)

### think() 核心逻辑
1. 历史对话 + 下一步提示词 → `llm.ask_tool()` 传入所有工具
2. LLM 返回要调用的工具名 + 参数
3. 保存到 `self.tool_calls` + 记录文本内容
4. `return bool(self.tool_calls or content)` 告诉上层

### act() 核心逻辑
1. 无工具调用:返回最后一条文本,本轮结束
2. 遍历 `tool_calls` 批量执行:
   - `await self.execute_tool(command)`
   - 工具结果封装为 `tool_message` 存记忆(Observe)
3. 收集所有结果 → 回到 BaseAgent 循环 → 下一轮 think

**完整闭环**:`think(选工具) → act(执行 + 存结果) → BaseAgent 循环 → 再次 think`。

## 4.4 Manus 第四层(成品)

继承 ToolCallAgent,预注册:
- **本地工具集**:PythonExecute / BrowserUseTool / StrReplaceEditor / AskHuman / Terminate
- **MCP 客户端**:`mcp_clients: MCPClients`

重写 `think()` 做浏览器特殊适配(运行时补充网页上下文,不影响全局)。

# 五、MCP 协议深度集成(关键)

## 5.1 MCPClients:MCP 服务统一管理器

**继承 `ToolCollection`**,完全复用本地工具管理逻辑,实现**远程 MCP 工具与本地工具无差别调度**。

**核心成员**:
- `sessions`:存多个 MCP 服务会话,支持同时连多台远程 MCP
- `exit_stacks`:异步资源栈,自动管理 MCP 连接释放,避免连接泄漏
- 构造函数初始化空工具列表,统一对外命名 `mcp` 保证版本兼容

## 5.2 MCPClientTool:远程工具代理层

所有 MCP 服务暴露的远程工具都被封装为 `MCPClientTool`,**统一继承 `BaseTool`**,对上层 Agent 完全透明:

1. 存储会话 / 服务 ID / 原始远程工具名
2. `execute()` 逻辑:
   - 校验 MCP 会话连接状态,未连接直接返回错误
   - 通过会话发起远程工具调用 `session.call_tool()`,传入参数
   - 解析服务返回的文本内容,封装为标准 `ToolResult` 返回
   - 全局捕获异常,统一包装工具执行错误

**设计价值**:**上层 Agent / LLM 不需要区分本地/远程工具**,调用逻辑完全一致,实现远程能力无缝扩展。

## 5.3 Manus 接入 MCP 流程

`connect_mcp_server()` 方法实现 MCP 服务动态接入:

1. 区分两种连接模式:
   - `use_stdio=true`:本地子进程 MCP 服务(stdio 通信)
   - `use_stdio=false`:网络 SSE 长连接远程 MCP 服务
2. 连接完成后,**自动提取该服务下全部远程工具实例**
3. **将远程工具批量加入** Manus 自身的 `available_tools` 工具集合
4. LLM 获取工具列表时,远程 MCP 工具会**和本地 Python / 浏览器工具一同下发**,推理时可自由选择调用

# 六、开源项目可复用优质底层设计

## 6.1 PythonExecute 代码沙箱(安全执行 4 大最佳实践)

提供**隔离、可控**的 Python 代码运行环境:

1. **进程隔离**:`multiprocessing.Process` 新开独立进程,代码崩溃不会破坏主智能体进程
2. **超时熔断**:设执行超时,超时直接终止进程,防止无限死循环卡死智能体
3. **受限安全环境**:拷贝纯净内置函数,屏蔽文件删除 / 系统命令等高风险权限
4. **输出捕获**:重定向标准输出,完整收集代码打印结果,统一返回执行成功标识 + 观测文本

## 6.2 BaseAgent 状态上下文管理器 `state_context`

基于**异步上下文管理器**实现安全状态切换:

1. 进入上下文前记录原始状态,切换为目标运行状态
2. 代码块执行异常时,自动将智能体状态置为 `ERROR`
3. `finally` 块强制恢复原始状态,杜绝状态卡死 / 状态错乱
4. 适用场景:工具执行 / 大模型推理 / MCP 连接等易出错流程

## 6.3 ToolResult 统一工具结果模型

标准化所有工具(本地 / MCP 远程)的返回格式,**支持多工具结果拼接**:

- 字段:`output` 正常输出 / `error` 错误信息 / `base64_image` 多媒体 / `system` 系统日志
- 重载 `__add__` 运算符,支持批量工具执行后**自动合并输出 / 错误 / 图片内容**
- 统一结果格式大幅简化记忆存储、结果回传给 LLM 的逻辑

# 七、Java 自研简化版 YuManus(实战)

## 7.1 数据模型定义

```java
public enum AgentState {
    IDLE,     // 空闲,可接收新任务
    RUNNING,  // 循环执行任务中
    FINISHED, // 任务正常完成
    ERROR     // 执行异常终止
}
```

**作用**:管控智能体生命周期,防止重复启动、异常状态未重置。

## 7.2 分层架构 4 层设计(对标 OpenManus)

| 层 | 作用 |
|---|---|
| **BaseAgent(抽象基类)** | Agent Loop 循环 / 状态管理 / 记忆 / 大模型基础封装 |
| **ReActAgent** | 拆分 Think / Act 两步,落地 ReAct 模式 |
| **ToolCallAgent** | 解析 LLM 工具调用、分发执行本地 / MCP 工具 |
| **YuManus(成品)** | 集成全部工具 / MCP 客户端 / 业务提示词 |

### BaseAgent 核心逻辑

```java
public void run() {
    // 1. 校验状态:仅 IDLE 允许启动
    // 2. 写入用户提问到记忆
    // 3. 切换状态为 RUNNING
    // 4. for 循环实现 Agent Loop,上限 maxSteps
    for (int step = 0; step < maxSteps && state != FINISHED; step++) {
        try {
            String stepResult = step();  // 抽象,子类实现
        } catch (Exception e) {
            state = ERROR;
        }
    }
    // 5. finally 资源清理
}
```

**关键点**:
- 模板方法模式:父类固定循环流程,子类只实现 `step()`
- 状态机 + 异常捕获:鲁棒性
- 抽象 `step()` 让 ReActAgent 专注 think/act

### ReActAgent 拆分

```java
abstract class ReActAgent extends BaseAgent {
    abstract String think();  // 推理
    abstract String act();    // 行动
    
    @Override
    String step() {
        String thought = think();
        if (thought.contains("DONE")) return thought;  // 终止
        return act();
    }
}
```

## 9.2 MVP 结构

```java
@Component
public class SimpleAgent {
    @Resource private ChatClient chatClient;
    @Resource private ToolCallback[] tools;
    private final List<Message> history = new ArrayList<>();
    private static final int MAX_STEPS = 10;
    
    public String run(String userTask) {
        history.add(new UserMessage(userTask));
        // Agent Loop
        for (int step = 0; step < MAX_STEPS; step++) {
            // think:AI 推理
            String response = chatClient.prompt()
                .messages(history)
                .tools(tools)
                .call()
                .content();
            history.add(new AssistantMessage(response));
            // 判断完成
            if (response.contains("DONE") || response.contains("任务完成")) {
                return response;
            }
        }
        return "达到最大步数";
    }
}
```
